# New Domain SEO + DR Playbook

> Checklist for setting up a fresh domain so crawl signals are clean and DR grows from day one.

## Prerequisites (before any link building)

- [ ] One canonical version (www or non-www), HTTPS, correct redirects
- [ ] Sitemap generated and submitted to Google Search Console
- [ ] Core pages live: Home, About, Contact, Pricing or FAQ
- [ ] Identical brand name, URL format, and contact info everywhere

## Indexable vs Indexed

- **Indexable** = Google is allowed to crawl it (no noindex, not blocked by robots.txt)
- **Indexed** = actually showing in Google. Check by searching the exact URL in quotes: `"https://yourdomain.com/page"`

## Baseline Brand Profiles (nofollow, trust signals)

Goal: help Google connect your brand to a niche. Save every profile URL you create.

| Platform | Setup | Notes |
|----------|-------|-------|
| **Crunchbase** | Company profile with name, description, site URL | Often ranks on brand searches, helps trust signals |
| **Facebook** | Page with clean username, "allow search engines" enabled | Link from website footer so Google discovers it |
| **Instagram** | Business account, 1+ post, 4+ followers, 4+ following | Link from site/directory listings to help indexing |
| **LinkedIn** | Company page, add to personal profile Experience | Wait a few days for association, link from footer, give Google 1-2 weeks |
| **Pinterest** | Business profile with website link | Easy credibility layer, good for visual brands |
| **X/Twitter** | Brand account, website in profile, professional account | Mention brand handle in posts instead of dropping links |

## DR Movers (dofollow or high-authority)

These move DR if set up cleanly. Avoid obvious two-way link swaps.

| Platform | How | Key Detail |
|----------|-----|------------|
| **Cal.com** | Profile with site link in about + real event types | Link cal page FROM a different domain than the one it links TO (avoid swap pattern) |
| **Figma Community** | Publish a file, add product URL as support contact | Link community page from another domain or indexed third-party page |
| **GitHub** | Public repo, website in repo About field | Link to repo from another site. Crawlers are mobile-first, link rendering may differ |
| **Topmate** | Profile with site link in socials/custom fields | Link topmate page from a different domain than main site |
| **Dribbble** | Job post with product link in listing | Costs money but page persists, strong domain |
| **Webflow Showcase** | Public profile, submit to showcase | Only if actually on Webflow |
| **TheHiveIndex** | Submit community, link to community landing page on your domain | Only if you have a real community. Good for early trust |
| **Algolia HN Mirror** | Post on HN, reference mirrored page | Small bonus, not core strategy |
| **10comwebdevelopment** | ABC-style exchange | Optional, not as clean as product-based links |
| **Portotheme** | Guest post or link insertion | Paid placement, keep it relevant and natural |

## Process for Each New Domain

1. Technical setup (canonical, HTTPS, sitemap, GSC)
2. Publish core pages (home, about, contact, pricing/FAQ)
3. Create all baseline brand profiles (save URLs in a tracking sheet)
4. Verify indexability - check robots.txt, no noindex tags
5. Wait for initial indexing (1-2 weeks)
6. Build DR mover links one at a time, spacing them out
7. For each DR link: ensure no direct two-way swap pattern
8. Monitor DR monthly via Ahrefs

## Anti-Patterns

- Building authority links before crawl signals are clean
- Two-way link swaps (A links to B, B links to A)
- Inconsistent brand name/URL/contact info across profiles
- Dropping links in every social post instead of brand mentions
- Skipping the baseline profiles and going straight to DR movers
