# Reports

Phase reports for multi-session builds. Each folder = one significant build session.

## Structure

```
reports/
  YYYY-MM-DD_slug/
    technical-report.md    ← Dev continuity (Claude, Nio, future sessions)
    plain-english-report.md ← Explain to anyone (Shawn, partners, audience)
```

## How Reports Get Created

- **Automatically** via the `/phase-report` skill when Claude completes a significant build
- **Manually** by asking Claude to "write a phase report" or "document this"
