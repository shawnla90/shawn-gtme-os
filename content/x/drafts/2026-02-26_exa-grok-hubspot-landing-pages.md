## Thread

### Tweet 1 (hook)

I automated personalized landing pages from terminal to CMS.

one command. 90 seconds. live in HubSpot.

no drag-and-drop. no CMS editor. no copy-pasting.

here's the stack and how it works 🧵

### Tweet 2

the problem:

building account-based landing pages meant researching a company, writing custom copy, formatting it, uploading to HubSpot. repeat.

45 min per page. fine for 5 accounts. nightmare for 50.

### Tweet 3

the pipeline:

Exa MCP → real-time company intel (firmographics, tech stack, news, hiring signals)

Python SDK → orchestrates the flow, formats research into a structured brief

Grok → generates personalized landing page copy from the brief

HubSpot CLI → pushes the finished page live. no browser needed.

### Tweet 4

the key: MCP servers let the model call Exa as a native tool.

it decides what to research and how deep to go. I'm not writing API calls - the agent is using the tool.

research quality goes way up when the model controls the queries.

### Tweet 5

what used to take 45 min per page now runs in under 2 min.

and the copy is better because every page references real things about the company - not "we help businesses like yours" filler.

### Tweet 6

the unlock isn't any single tool.

it's: MCP for research → SDK for orchestration → model for generation → CLI for deployment.

no GUI in the loop. the whole GTM workflow lives in the terminal.

if you're still building landing pages one at a time you're spending hours on what should be a script.
