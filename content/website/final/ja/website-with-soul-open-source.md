## 三部作が完結した

今日[website-with-soul](https://github.com/shawnla90/website-with-soul)をGitHubにプッシュした。方法論から始まり、インフラに進み、プロダクトで終わる三部作の、3番目にして最後のリポジトリだ。

[recursive-drift](https://github.com/shawnla90/recursive-drift)はAIと思考するためのオペレーティングシステムだった。自分の声を失わずにClaude Codeをコラボレーターとして使う方法。6つの状態、自己読み取りフィードバックループ、APIキー不要。

[context-handoff-engine](https://github.com/shawnla90/context-handoff-engine)は配管だった。Claude Codeが毎セッションをゼロから始めないための6層のコンテキストインフラ。パラレルセーフなハンドオフ、構造化メモリ、自己改善ループ。

website-with-soulは、すべてが着地する場所だ。最初の2つのリポジトリのすべては、これを機能させるために存在する。自分らしく聞こえ、検索でランクインし、AIシステムに情報を供給し、時間とともに複利するリアルなウェブサイトを構築する。

## 中身

2つのものが入っている。

**動作するスターターテンプレート。** クローンして、`npm install && npm run dev`を実行すれば、15分でサイトが完成する。ダークターミナル美学、markdownブログ、キーワードRAG付きAIチャットウィジェット、プロキシパターンのPostHogアナリティクス、AIクローラー許可リスト付きサイトマップ、OG画像生成、RSSフィード、セキュリティヘッダー。すべてNext.js 15とTailwind v4上で動く。

**32章のプレイブック。** 空のディレクトリからコンテンツマシンまでの全工程を4フェーズでカバー。

フェーズ1はビルド。スターターテンプレートのすべてのファイルを解説する11章。セットアップ、ブログシステム、チャットウィジェット、アナリティクス、サイトマップ、OG画像、RSS、セキュリティヘッダー、パフォーマンス、デプロイ、Claude Codeセットアップ。

フェーズ2は魂を入れること。ボイスDNA、アンチスロップパターン、コンテンツ整理、ブログワークフロー、マルチプラットフォーム配信、SEOパイプライン。ほとんどの人がスキップする部分であり、最も重要な部分だ。

フェーズ3はオーガニックな成長。配信マトリックス、Reddit戦略(AIインデックスコンテンツに関するGEO理論を含む)、LinkedInビルダーボイス、Xスレッドフォーマット。有料広告なし。ウェブサイトがハブで、ソーシャルプラットフォームは配信チャネル。

フェーズ4はスケーリング。モノレポアップグレード、自律ブログパイプライン、cron自動化、Claude Codeエージェントシステム、Cloudflare Pagesへの移行。

## 人々が間違える部分

みんな「パブリックにビルドしよう」と言う。インフラを所有することについてはほとんど誰も語らない。

LinkedInに毎日1年間投稿して、リアルなオーディエンスを構築できる。LinkedInがアルゴリズムを変更する。リーチが一晩で60%減る。賃貸の土地の上に建てたということだ。

ウェブサイトは自分が所有するものだ。すべてのブログ記事が複利する。すべてのページがインデックスされる。すべてのAIクローラー(GPTBot、ClaudeBot、PerplexityBot)がコンテンツを消費し、ソースとしてあなたを参照し始める。これがプレイブックのGEO理論だ。もはやGoogleだけに最適化しているんじゃない。Googleに取って代わりつつあるAIシステムに最適化しているんだ。

ソーシャルプラットフォームは配信チャネルだ。自分が所有するものにトラフィックを戻す。ウェブサイトだ。

## なぜオープンに構築したか

3つのサイトがまさにこのスタックで動いている。[shawnos.ai](https://shawnos.ai)、[thegtmos.ai](https://thegtmos.ai)、[thecontentos.ai](https://thecontentos.ai)。同じコードベース、同じプレイブック、同じボイスシステム。スターターテンプレートは本番コードから直接抽出したものだ。

これをプロプライエタリにしておくこともできた。有料コースやSaaS製品にすることもできた。しかしrecursive-driftの全体の命題は、方法論は共有されることで改善するということだ。使って、フィードバックをくれる人が増えるほど、良くなる。

だからMITライセンスだ。好きに使ってほしい。

## 始め方

どれだけ深く入りたいかによって3つの道がある。

**15分。** スターターをクローンし、コンテンツをカスタマイズし、Vercelにデプロイ。ウェブサイトの完成だ。

**1時間。** フェーズ1のプレイブックに沿って進む。すべてのファイルを理解し、カスタムドメインでライブに。

**1日。** 4フェーズすべてを通して作業する。最後には、ボイスシステム、配信パイプライン、SEO戦略を備えたコンテンツマシンが完成する。
