## 时间节点

Clay 今天发布了新定价。不是微调，是全面重构。新计划名称、新货币体系、功能在层级之间重新分配。他们甚至公开了内部定价备忘录 - 要么是非常自信，要么是精心策划。可能两者都是。

我在 Clay 里日常构建已经超过一年了。表格、Claygents、HTTP API 集成、数据充实架构。我在 [Clay Wiki](https://thegtmos.ai/clay-wiki) 里记录了60多种模式。我不是 Clay 合作伙伴，不是推广联盟，你买 Clay 我不拿钱。

所以这是一个每天真正使用这个工具、且没有任何利益倾向的人的拆解。

## 到底改了什么

旧模式很简单。一种货币：credits。你在 Clay 里做的一切都消耗 credits。数据充实、AI 调用、HTTP 列、全部。计划分为：Free、Starter（$149）、Explorer（$349）、Pro（$800）、Enterprise（定制）。

新模式把单一货币拆成了两种：

**Data Credits** 用于数据充实，也就是市场里的那些。Apollo 查询、邮箱查找、公司信息数据。Clay 说他们把前20种最常用充实的数据成本降低了50-90%。

**Actions** 用于其他一切。每次 Clay 执行编排工作，就是一个 Action。充实运行、Claygent 调用、HTTP API 执行、CRM 推送、webhook 触发。公式和导入不算。但基本上任何涉及 Clay 代你执行工作的操作，现在都消耗一个 Action。

新层级：Free、Launch（$185/月）、Growth（$495/月）、Enterprise（定制）。

## 没人给你做的并排对比表

**旧世界：**

- Starter：$149/月，2,000-3,000 credits，无 CRM，无 HTTP API
- Explorer：$349/月，10,000-20,000 credits，HTTP API + webhooks，无 CRM
- Pro：$800/月，50,000-150,000 credits，CRM 集成，5个团队席位

**新世界：**

- Launch：$185/月，15,000 Actions，2,500-20,000 Data Credits
- Growth：$495/月，40,000 Actions，6,000-100,000 Data Credits，CRM 同步，HTTP API，webhooks，Web Intent
- Enterprise：定制，100K+ Actions，100K+ Data Credits，数据仓库同步，SSO，批量充实

入门价从 $149 涨到 $185。但每个层级的功能集发生了显著变化。不看每个层级现在包含什么，就没法做简单的美元对比。

## 谁赢了

### 只为 CRM 同步付 $800 的团队

这是最明显的赢面。CRM 集成（HubSpot、Salesforce）以前需要 $800/月的 Pro 计划。现在在 $495 的 Growth 就有了。如果你付 $800 主要是因为需要 Clay 和 CRM 同步，切换后每月省 $305。

一年 $3,660。对很多小团队来说，这很可观。

### 重度数据充实用户

如果你跑大量数据充实，50-90% 的数据成本降低是实打实的省钱。Clay 内部备忘录说他们预计短期会因此损失收入。这意味着节省是真实的，不是营销数学。

我在 [Clay Wiki](https://thegtmos.ai/clay-wiki/account-first-enrichment) 里教的以账户为先的充实模式本身就通过在公司层面去重来节省 credits。在好的架构之上再叠加更便宜的单次充实成本，意味着你的单位经济效益双重改善。

### 想要 Web Intent 和 Ads 的构建者

Clay 把较新的产品（Web Intent 信号、Clay Ads）打包进了 Growth 层级。这些以前是企业专属或单独付费的附加功能。如果你在构建基于意图的工作流，Growth 计划现在给了你更大的操作空间。

## 谁输了

### 依赖 HTTP API 的 Explorer 用户

这是最痛的变化，但没多少人在讨论。

旧的 Explorer 计划在 $349/月就给你 HTTP API 访问。HTTP 列是构建者连接 Clay 到外部服务、自定义 API 和自动化工具的方式。这是构建者的计划。

新定价下，HTTP API 需要 $495/月的 Growth 计划。仅仅为了继续做你已经在做的事，就多了 $146/月。更致命的是：HTTP API 调用现在也消耗 Actions。以前是免费的（除了 credit 成本）。现在是一个计量资源。

如果你在 Explorer 层级围绕 Clay 的 HTTP 列构建了整个 GTM 技术栈，这个变化是双重打击：更高的计划费用加上每次 API 调用的新消耗。

### 不需要 15,000 Actions 的轻度用户

旧的 Starter 计划 $149/月确实很轻量。导入一些联系人，跑几次充实，导出。不需要 API 访问，不需要 CRM 同步，就是基础的 Clay。

新的 Launch 计划 $185/月给你 15,000 Actions。如果你每月只跑 2,000 次充实，你多花了 $36 买了永远用不到的容量。Clay 说90%的客户不会用完 Actions 配额。但这意味着10%会。而轻度用户在为更高的底线买单。

### 喜欢简单定价的人

一种货币很干净。你知道东西多少钱。双货币增加了认知负担。现在你要同时考虑 Data Credits（数据充实花多少钱）和 Actions（Clay 的编排花多少钱）。两个数字要追踪，两个上限要监控，两样东西可能在工作流进行中用完。

对有经验的构建者来说这可以应付。对刚开始用 Clay 的团队来说，这是在你还没来得及评估工具是否好用之前又多了一个要学的东西。

## 构建者到底该怎么做

### 如果你在旧计划上：别慌，算账

Clay 会保留现有计划。你不必切换。但你有到2026年4月10日的窗口，可以在旧计划之间做一次性切换，如果另一个旧层级对你更合适的话。

拉出你最近3个月的 credit 使用记录。看看你实际消耗了多少 vs. 你付了多少。然后对照新层级建模。每个团队的算法都不一样。

### 如果 HTTP API 是你的命脉：仔细算数

统计你每月发起多少次 HTTP API 调用。在新模式下，每次调用就是一个 Action。Growth 给你 40,000 Actions。如果你跑 5,000 次 HTTP 调用加 10,000 次充实再加 Claygent 调用，加起来看 40,000 Actions 够不够。Actions 计量器包含一切，不只是 HTTP。

### 如果你是 Clay 新手：从 Growth 开始

Growth 计划 $495/月是认真构建者的新甜蜜点。CRM 同步、HTTP API、webhooks、Web Intent、40,000 Actions、最多 100,000 Data Credits。以前光 CRM 这一项就需要 $800 的 Pro 计划。

如果你在构建生产级 GTM 工作流，Growth 是工具所在的层级。

### 如果你几乎不用 Clay：考虑你是否需要它

这话来自一个每天用 Clay 的人。如果你每月跑 1,000-2,000 次充实，没有 API 集成，没有 CRM 同步，你现在每月花 $185 买的本质上是一个带充实列的高级电子表格。这可能仍然值得。但也可能值得直接通过 Apollo 或 Prospeo 的 API 跑这些充实，省下平台费用。

Clay 的价值在于编排，不在于单个充实。如果你不用编排，你就是在为平台付费但没用到平台。

## 更大的图景

Clay 公开了内部定价备忘录。他们明确说预计短期收入下降约10%。这是一个真实的赌注。他们在说：我们现在会少赚，因为我们相信更便宜的数据加上更可及的功能会让人们构建更复杂的工作流，从而推动 Actions 消耗。

这是一个平台策略。他们想让你深入 Clay。更多 Claygents、更多 HTTP 列、更多 CRM 同步、更多自动化。每一个都消耗 Actions。数据更便宜了，但编排现在是计量的。

这无所谓好坏。这是一个商业模式的转变，从"为数据付费"到"为平台为你做的事付费"。理解这个转变，决定了定价是在为你服务还是在对你不利。

## 我在 Clay Wiki 里更新什么

我正在 [Clay Wiki](https://thegtmos.ai/clay-wiki) 里添加一个完整的定价模型条目，涵盖新的双货币体系、层级对比和迁移决策框架。现有的 [credit 系统指南](https://thegtmos.ai/clay-wiki/credit-system) 对于理解单个充实如何消耗 credits 仍然适用。新条目覆盖的是在此之上的结构性变化。

这也是为什么免费审计现在比上周更重要。如果你在评估是否要切换计划、留在旧计划、还是重构 Clay 架构以优化新模式，[那30分钟的对话](https://shawnos.ai/blog/before-you-hire-a-clay-agency) 可能帮你省下数千美元的错误支出。

## 我的观点

Clay 正在变得更像一个平台，而不是一个数据工具。定价反映了这一点。如果你是一个把 Clay 当作编排基础设施（CRM 同步、HTTP API、Claygent 工作流、多步充实）的构建者，新定价可能对你有利。更便宜的数据，更低层级就能获得更多功能。

如果你是一个只想要充实而不需要平台的轻度用户，底线刚刚提高了，复杂度也增加了。

如果你在 Explorer $349 并且重度使用 HTTP API，切换前先算好你的数字。你的旧计划可能是更好的选择。
