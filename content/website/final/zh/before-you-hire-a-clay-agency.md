## 我一直被问到的问题

每周都有人私信我同一个问题。措辞不同，意思一样。

"我应该雇一个 Clay 代理商吗？"

"[某代理商]靠谱吗？"

"我们在 Clay 上每月花 $X，我完全不知道有没有价值。"

我不开 Clay 代理商。也没打算开。但我从早期就深入 Clay - 建表、调试 Claygent 提示词、写 HTTP API 集成、在 [Clay Wiki](https://thegtmos.ai/clay-wiki) 记录补全瀑布流。60多个条目，还在增长。

所以人们来问我。我给出的建议总是一样的：在你签任何东西之前，先审计你已有的东西。

## 审计清单

以下是我和每个来问的人过的内容。五个问题，告诉你代理商合作到底是真的有帮助，还是只是又加了一层不透明。

### 1. 你有多少个工作空间在运行？

大多数团队不知道。Clay 工作空间增长得很快 - SDR 团队一个，代理商建了一个，某个试点项目没人关掉又一个。每个工作空间都有自己的积分池、自己的表、自己的"事实版本"。

如果你一只手数不过来你的工作空间，也说不清每个是干什么的，那就是第一个红旗。光是合并就能每年省下数千美元。

### 2. 你的 GTM 工程师在高效使用积分吗？

积分是 Clay 的隐性成本。每次补全、每次 Claygent 调用、每次 API 查询都在消耗积分。我见过团队一个月烧掉50,000积分，就因为没人设置条件逻辑来跳过已经有的补全数据。

解决方法几乎总是一样的。补全之前先检查数据是否已存在。把结果缓存到数据湖里，而不是跨活动重复补全同一批线索。设置积分追踪，让你知道每个表运行一次要花多少。

我写了一篇完整的 [GTM 工具积分透明度拆解](https://thegtmos.ai/how-to/credit-transparency-gtm-tools)，如果你想看实施细节的话。

### 3. 你是不是买了比需要更多的积分？

Clay 的定价层级改过好几次了。有些团队还在用两年前合理但现在不合理的旧方案。另一些买了企业版但从没用过那些功能。

打开你的账单页面。对比你的月度积分消耗和计划分配。如果你持续只用了付费额度的40%，你在多付。如果你每个月都到上限还在买超额积分，你可能需要换一个层级 - 或者提高表的效率。

### 4. 你理解供应商锁定吗？

如果代理商构建了你的 Clay 表，谁拥有它们？合作结束时会怎样？你的团队能维护和修改这些表吗，还是它们变成了只有代理商才能理解的黑盒？

我见过公司每月付代理商 $5,000 来运行他们其实可以用一周培训就内部维护的表。知识转移从未发生，因为合同里没写。

在和任何代理商签约之前，问清楚：
- 谁拥有工作空间和表？
- 你的团队会得到文档和培训吗？
- 如果换供应商，你能导出并重建吗？
- 他们用的自定义集成是不是只有他们能维护？

### 5. 有人在监控表的效率吗？

一张六个月前正常工作的 Clay 表，现在可能有30%的补全在返回空值。数据供应商会变，端点会下线，在 GPT-4 上好用的提示词换了最新模型可能出垃圾。

必须有人盯着命中率、错误率和每行成本。如果没人盯，你就是在盲飞 - 不管你有没有代理商。

## 免费审计

30分钟。没附加条件。没推销 PPT。只是一次屏幕共享，我们一起看你的 Clay 设置。

我会看什么：
