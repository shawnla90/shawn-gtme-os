defaulting to Opus for everything is expensive and it makes your results inconsistent. I know because I did it for two months straight.

the counterintuitive part: throwing more model at a problem doesn't always improve the output. sometimes it degrades it. Opus has a tendency to over-reason simple tasks. ask it to extract a phone number from a string and it'll give you three paragraphs about edge cases.

here's how I actually route now across a monorepo with 50+ skills.

---

**Haiku** . for anything that's fast, cheap, and structured

this is my workhorse for:
- classification tasks ("is this company in ICP or not" → yes/no + reason)
- structured extraction where the schema is tight (pull name, title, company from a block of text)
- first-pass filtering before a heavier model runs
- reformatting or light transformation (convert this JSON shape to this other JSON shape)
- anything inside a loop that runs 100+ times

example prompt pattern:

```
You are a classifier. Output only valid JSON.
Input: {raw_text}

```

Haiku on this? Instant. Accurate. Costs basically nothing. If I ran Opus here across 500 rows it'd be 20x the cost with zero improvement in accuracy.

---

**Sonnet** . my default for most real work

I reach for Sonnet when:
- the task requires reasoning but not extended research
- I'm writing code that needs to be correct but isn't architecturally complex
- prompt needs nuance but the output is bounded (write a 3-sentence cold email variation using these signals)
- I need tool use / function calling with moderate complexity
- anything that'll be shown to a user directly

this is where 80% of my calls land. it's fast enough to feel snappy, smart enough to handle messy inputs, and cheap enough that I'm not watching the meter.

---

**Opus** . for genuinely hard problems

I use Opus specifically when:
- I'm architecting something new and I want the model to push back on my approach
- the context is huge and the reasoning chain actually matters (debugging a multi-file system, reviewing a full workflow design)
- I'm writing a meta-prompt or a skill that other prompts will use downstream
- I'm stuck and Sonnet gave me a mediocre answer twice

what I don't use Opus for:
- first drafts
- any task I can clearly specify up front
- loops or batches
- classification

the tell that you're over-routing to Opus: you're hitting limits fast and your results feel mushy. Opus on a simple task over-explains, adds caveats, second-guesses your constraints. it's optimized for ambiguity. if your task isn't ambiguous, it works against you.

---

**the routing heuristic I actually use**

ask yourself: "if I gave this to a smart junior who just needed clear instructions, could they do it?"

→ yes, schema is clear, input is structured = Haiku
→ yes, but needs judgment / context = Sonnet
→ no, I'd have to think through it with them = Opus

for agentic stuff: start with Sonnet as your orchestrator, drop to Haiku for sub-tasks, only escalate to Opus when the orchestrator hits a genuine decision point it can't resolve.

---

costs roughly (at API rates, varies by tier):
- Haiku: ~$0.25/M input tokens
- Sonnet: ~$3/M input tokens
- Opus: ~$15/M input tokens

if you're running 10k classification calls a day, that's the difference between $2.50 and $150. same accuracy. not a drill.
