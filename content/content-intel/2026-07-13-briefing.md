### REDDIT Version

how to unvibe code a vibe coded app (step by step audit loop)

shipped fast, broke in prod. sounds familiar.

vibe coding gets you to v1 in a weekend. the agent writes everything, you approve changes, the app works. until someone adds a user, or you try to add a feature, or you onboard a second dev and nobody knows what anything does.

here's what I do when I inherit a vibe coded codebase (including ones I built myself three weeks ago and forgot):

**step 1: schema read first, touch nothing**

drop into your DB and actually read what exists. don't assume you know. ask the agent to map every table, every foreign key, every nullable column that probably shouldn't be.

prompt that works:
```
read every file in /db or /prisma/schema (wherever your ORM lives).
list every table, explain what it stores, flag anything that looks orphaned
or inconsistent. do not fix anything. just map it.
```

you'll find tables with no references, columns named `temp_fix`, and three different user ID formats coexisting. cataloging before touching is the whole game.

**step 2: trace the auth flow**

this is where vibe coded apps break hardest. the agent wired auth but nobody knows if it's enforced at every route.

ask the agent to trace the full auth flow on your three most sensitive endpoints:
- where is the session/token actually validated
- what happens if it's missing or expired
- does middleware run before or after you think it does

```
trace the auth check for [endpoint]. follow the middleware chain.
show me every place this request could succeed without a valid session.
```

nine times out of ten there's a gap. find it before your users do.

**step 3: test coverage loop (targeted, not everything at once)**

vibe coded apps have zero tests. or they have AI-generated tests that test nothing real.

don't ask the agent to write tests for everything at once. you'll get 200 green tests and zero confidence.

pick your five highest-risk paths (auth, payments, data mutation, anything touching PII) and run this per path:

```
for [specific function or route]:
1. explain what this code does in plain english
2. list the 3 most likely failure modes
3. write tests that specifically target those failure modes
```

five routes, five runs. now your test suite means something.

**step 4: dead code pass**

vibe coded apps accumulate scaffolding fast. the agent tries something, it half-works, the agent moves on. now you have three versions of the same function.

```
scan the codebase for unused functions, dead imports, and duplicate logic.
do not delete anything. list it and explain why each one looks unused.
```

human reviews the list, approves the cuts. never let the agent delete without sign-off on this pass.

**step 5: make it write the README from what actually exists**

not from your original intent. from the code it finds.

this surfaces every place the implementation diverged from the plan. and now you have documentation that's actually true.

---

the full loop runs 2-3 hours on a mid-size vibe coded app. you come out knowing what you actually built, where the holes are, and what you can hand to someone else without a 45-minute explanation session.

ship fast, audit after. that's the cadence.
