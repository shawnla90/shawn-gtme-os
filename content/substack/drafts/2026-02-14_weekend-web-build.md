## Post Body

## the plan

I was going to tell you the Cursor story today. the moment a code editor became an operating system. that's still coming. monday.

but the system needs a home on the web first.

## why now

the content OS lives in a repo. LinkedIn, X, Substack, Reddit. four platforms, four playbooks, four pipelines. it works. it's modular. it's growing.

but none of those platforms are mine.

LinkedIn could change their algorithm tomorrow. Substack could sunset a feature. every piece of content I create lives on rented land. and I've been building a system designed to outlast any single tool.

the system itself doesn't have its own address. this weekend that changes.

## the weekend sprint

I'm learning web hosting from scratch. and I do mean scratch.

two weeks ago I didn't know what a git repo was. now I'm about to register a domain, set up DNS through Cloudflare, scaffold a Next.js project on Vercel, and build a site that matches the terminal aesthetic you've seen across these posts. dark background. green accents. monospace type. the AI/os brand as an actual website.

the stack:

- **osailabs.ai** as the domain
- **Vercel** for hosting (auto-deploys when I push to GitHub)
- **Next.js** for the framework
- **Cloudflare** for DNS and CDN
- **markdown files** for the blog (same pattern as everything else in the repo)

no Squarespace. no drag-and-drop. the whole site lives inside the repo. when I push a blog post, it deploys automatically. zero extra steps.

## what this unlocks

the website becomes the canonical home. blog posts live there first. Substack becomes a syndication channel. which means:

- SEO works (my domain, my ranking)
- I control the design and the experience
- content lives in one place and distributes everywhere
- the repo becomes the operating system for the entire web presence. not just content. everything.

this isn't a redesign project. this is the next layer of the system.

## the learning edge

I have no idea how Vercel works. I've never registered a domain. I've never configured DNS. I've never touched Next.js.

two weeks ago I was an SDR who learned GTM from grinding Salesforce. now I'm about to build and deploy a website from a code editor.

the system doesn't require you to already know everything. it requires you to be willing to learn the next skill. and the next one.

this weekend. osailabs.ai. from nothing to deployed.

monday I'll tell you the Cursor story. today I just wanted to share the plan while it's still a plan. because by monday it'll be a build log.
