# from tabs to terminals

> **Platform**: LinkedIn
> **Pillar**: Building & Sharing
> **Date**: 2026-04-01
> **Status**: draft v5 (learning journey + narrative restructure)
> **Source**: Substack - from tabs to terminals

---

a few months ago my GTM workflow was 14 browser tabs. over $2K/mo. progress bars I was watching. dashboards I was alt-tabbing between.

I was spending more time switching between tools than doing the work the tools were supposed to help with.

but I didn't just decide to switch to terminals one day. there was a learning curve.

started on Cursor. burned through the plan in a week using only Opus. tried going direct to APIs. spent $600 on Opus calls through OpenClaw. then realized everything I was doing could be done on Claude Code. same model. but inside a terminal that reads my files and runs commands.

even then it wasn't instant. had to learn context engineering. how tokens work. went through the MCP phase where I loaded every integration I could find. fifteen MCP servers when I needed three. sessions slowed down. limits disappeared.

the real unlock: most GTM tools have a CLI that does the same thing as the MCP without the token overhead. install, authenticate, run a command, move on.

five months in now. every month the sessions get tighter.

right now I have 4 terminals open. one scraping 10K followers via Apify CLI. one enriching through Apollo. one pushing to Supabase. one writing this post. none blocking each other. all on one Mac Mini.

the skill trap: everyone is loading dozens of skills and MCP servers and wondering why sessions burn through limits. the skill is using the tool itself. knowing how to direct it. a clean CLAUDE.md and organized repos do more than 40 custom skills.

I've been getting DMs from SDRs, students, people trying to learn GTM. the first thing they hit is a paywall from someone who built one project six months ago and decided to teach.

so I open sourced a 10-chapter playbook. the gtm-coding-agent repo. one of eight repos. all products of real builds. MIT licensed. free.

the person selling you an AI course in 2026 probably used Claude to write the course materials. you're paying a middleman to resell the output of the tool they're teaching you to use.

I will never sell a course. I'm building my own product and sharing everything along the way. fork it, lift from it, make it yours.

the first thing you build will be ugly. the tenth one will surprise you.

no gatekeeping ✌

shawn ⚡ the gtme alchemist 🧙‍♂️
